### DB 미니프로젝트 정리
- 데이터베이스 구축대상 선정
- 조직 또는 업무 등 데이터베이스 구축 대상으로 선정된 범위내에서 일어나는 일들을 시나리오로 작성

1. 가계부 요구사항 정의
- 회원정보를 기록한다.
- 회원의 계좌 정보들을 기록한다.
- 입/출금을 하게되면 회원의 계좌에 바로 기록이 된다.
- 입/출금 입력시 금액, 사용계좌, 날짜, 사용자(사용자의 경우 접속 ID로 자동 반영), 사용장소, 위치, 사용목적 등을 입력한다.
- 사용목적 칼럼의 경우 정해진 카테고리 내에서 선택할 수 있도록 한다.
(추후 카테고리별 사용 금액 확인 용도)
- 선택한 계좌들의 전체 금액을 출력할 수 있다.
- 원하는 기간별 사용 금액을 출력할 수 있다.

- 목표금액을 기록한다. 목표금액은 월별 최대로 사용하려는 금액, 1년후 모으려는 금액등
- 내가 지출한 금액이 목표한 금액보다 많은지 적은지
- 월평균 사용량보다 지출이 많은 달 출력
- 기간별 입출금 내역 확인 가능
- 내가 쓰는 금액이 표본에 비해 얼마나 쓰는건지
- 카테고리별 지출 형태를 파악할 수 있다.

저축목표가 따로 필요하진않은거 같다? -> 이름도 다른걸로 바꺼보고 테이블명 처럼

### 테이블 설계
- 회원(User) 
ID(ID)	varchar2(12)	Primary key
패스워드(Password)	varchar2(15)	not null
이름(U_Name)	varchar2(10)	not null
성별(Sex)	char(1)	not null	default 'M'

- 계좌(Account)
계좌번호(A_Number)	varchar2(20)	primary key
계좌명(A_Name)	varchar2(40)	not null
잔액(Balance)	number(12, 0)	default 0    <<<<< 
ID(ID)	varchar2(12)	FK
          
- 입출금내역(Transaction)
